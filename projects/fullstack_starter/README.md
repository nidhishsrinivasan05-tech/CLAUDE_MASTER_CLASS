# Fullstack Starter

This project is a blueprint that shows the relationship between:
- frontend
- backend
- auth
- protected data
- dashboard growth path

It is intentionally minimal so you can grow it yourself.

---

## Core idea

A full-stack app usually means:
- a frontend interface
- a backend API
- some persistent data layer
- authentication
- deployment

This starter gives you the frontend/backend split first.

---

## Recommended growth path

1. run backend and verify `/health`
2. run frontend
3. add register and login to backend
4. create login form in frontend
5. store token after login
6. call protected backend route
7. build dashboard page
8. replace starter state with API-driven state

---

## Why this matters

Many learners understand backend alone or frontend alone.
The difficulty begins when both sides must agree.

That agreement includes:
- route paths
- request shapes
- response shapes
- error handling
- token passing

This is why full-stack development is partly about contracts, not just code.
