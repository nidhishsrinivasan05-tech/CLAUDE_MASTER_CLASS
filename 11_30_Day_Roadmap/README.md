# 11. 30 Day Roadmap

## Purpose of this roadmap

A roadmap turns a pile of knowledge into action.
Without action, even great notes become shelf decoration.

This plan is designed to help you move from reading to building.

---

## Week 1: Foundations and setup

### Day 1
Read:
- `01_Foundations`
- `02_Setup_and_Tooling`

Do:
- install tools
- verify Python, Node, and Git
- create one fresh project folder

### Day 2
Build:
- one FastAPI `/health` route

Learn:
- how the server starts
- how routes work
- how to test in browser or Postman

### Day 3
Add:
- one POST route
- one request model

Learn:
- request bodies
- validation
- JSON responses

### Day 4
Build:
- notes API

Learn:
- CRUD basics
- error handling
- 404 patterns

### Day 5
Read:
- `04_Prompt_Engineering`

Do:
- rewrite three weak prompts into stronger ones

### Day 6
Read:
- `05_CLAUDE_md_Project_Brain`

Do:
- write your first `CLAUDE.md`

### Day 7
Review:
- all work so far
- clean the folder
- commit progress with Git

## Milestone at the end

By Day 30, you should have:
- one well-documented repo
- one working auth-enabled backend
- one set of reusable prompt templates
- one monetizable app idea
- one stronger developer workflow than before

That is a serious upgrade from simply “watching a video”.
